   - Do not make the same RentCast request multiple times during the same screen load
   - Avoid re-fetching just because a component re-renders
   - Avoid duplicate calls on refresh if recent data is already available

   - Load only the minimum data needed for the initial result

   - If the user reloads the page shortly after a search, try to show cached data instead of immediately calling RentCast again
   - Add an explicit Refresh Data action if needed, rather than auto-refreshing everything

   - If any search suggestions or live validation exist, debounce them properly
   - Do not let rapid clicks or repeated submit actions create multiple identical requests
   - Disable or guard repeat submissions while a request is already in flight

   - Centralize RentCast requests in one service/module
   - Make it easy to see where requests are happening

   - If one user search can trigger multiple endpoint calls, reduce that where possible

9. Add error handling and fallbacks
   - Handle rate limits and API failures gracefully
   - If cached data exists and the API fails, show cached data with a note that it may not be the newest
   - Avoid retry loops that could accidentally increase cost

10. Keep the UX clean
   - The user should still feel the app is responsive
   - Show loading states clearly
   - Indicate when data is cached vs freshly pulled if helpful
